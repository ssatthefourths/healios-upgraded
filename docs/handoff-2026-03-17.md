### 3. Multi-Currency
- **Frontend**: `CurrencyContext.tsx` calls `/currency` on first session load, gets live rate, stores user preference
- **Worker**: `/currency` fetches from `api.frankfurter.app` (free, no key), Cloudflare edge-caches for 1 hour, falls back to hardcoded rates on failure
- **Checkout**: Converts GBP D1 prices to user's selected currency using the same live rates before sending to Stripe
- Supported: GBP, ZAR, USD, EUR, CAD, AUD

### 4. Payment Flow (end-to-end)
```
Cart → Checkout.tsx → supabase.functions.invoke('create-checkout-session')
     → functions bridge → POST /checkout-session (worker)
     → D1 price validation + live currency conversion
     → Stripe Checkout Session created
     → Redirect to Stripe hosted page
     → User pays
     → Stripe fires webhook → POST /stripe-webhook (worker)
     → HMAC verified → Order created in D1
     → Stock decremented in D1
     → Confirmation email via Resend
     → User redirected to /checkout?success=true&session_id=...
```

## Local Development

```bash
git clone https://github.com/ssatthefourths/healios-upgraded.git
cd healios-upgraded
npm install
npm run dev
# → http://localhost:5173
```

The frontend defaults to the production worker (`https://healios-api.ss-f01.workers.dev`) — no `.env` file needed for basic local development.

**To run the worker locally:**
```bash
cd worker-api
npm install
npx wrangler dev
# → http://localhost:8787
```
Then create `.env` in the root:
```
VITE_CF_WORKER_URL=http://localhost:8787
```

## Architecture Decisions

| Decision | Rationale |
|----------|-----------|
| Supabase removed, re-exported as cloudflare | Kept all 75 import paths unchanged — zero component edits needed |
| GBP as D1 base currency | Products priced in GBP; converted to display currency on read |

| `public/_redirects` | Required for Cloudflare Pages SPA routing |

## Cloudflare Auto-Deploy — How It Works

**Cloudflare Pages (frontend):**
- Connected to GitHub repo via Cloudflare Dashboard → Workers & Pages
- **Every push to `main` automatically triggers a new build and deployment**
- Build takes ~60–90 seconds
- Find the live site: Cloudflare Dashboard → Workers & Pages → `healios-upgraded` → the deployment URL shows at the top
- Current URL: `https://healios-upgraded.pages.dev`

**Cloudflare Worker (backend API):**
- **Does NOT auto-deploy** — you must run `cd worker-api && npx wrangler deploy` manually
- The worker is separate from Pages; it's deployed to `healios-api.ss-f01.workers.dev`
- To set up GitHub Actions auto-deploy for the worker, add a workflow file that runs `wrangler deploy` on push to main with a `CLOUDFLARE_API_TOKEN` secret
